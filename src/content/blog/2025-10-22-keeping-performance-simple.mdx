Color is the first thing people notice and the last thing they forget. At Prism Studio, we treat color not as decoration but as architecture — a structural element that guides attention, creates hierarchy, and communicates emotion.

## Building a color system

Every Prism project starts with a color audit. We analyze the brand's existing palette, test it against accessibility standards, and map it into a systematic scale.

Our approach uses CSS custom properties organized into three layers:

**Primitive colors** — The raw palette values. These rarely appear in component code directly.

**Semantic tokens** — Meaningful names like `--color-text`, `--color-bg-surface`, `--color-brand`. These are what components actually reference.

**Theme assignments** — The bridge between primitives and semantics, allowing the same component to render in light mode, dark mode, or any custom theme.

## Why systems matter

Without a color system, design drift is inevitable. Different pages start using slightly different shades. Accessibility gets inconsistent. Dark mode becomes a nightmare.

With a well-structured system, every color decision is intentional and every theme change is a variable swap — not a redesign.

## Dark mode is not an afterthought

We design for both light and dark themes from day one. This means choosing colors that maintain sufficient contrast ratios in both contexts, and testing every component in both modes before shipping.

The result is a cohesive experience regardless of user preference — and a codebase that makes theme customization trivial.
